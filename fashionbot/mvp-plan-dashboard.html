The Five Claws

Each claw is a Maritime agent with a specific role in the pipeline

🗣️

Communication Claw

MVP

The nervous system — routes messages between all claws and interfaces with the team via Telegram.

  • Route inter-claw messages via Telegram channels
  • Send status alerts and availability notifications
  • Parse user commands (/status, /approve, /scrape)
  • Compile daily summary reports with P&L
  • Manage alert priority levels (critical/high/normal)
Telegram Bot API Webhook Router DB Read
🔍

Research Claw

MVP

Curates and validates item lists — confirms links, colors, sizes, and suggests trending alternatives.

  • Receive manual item list from user (up to 10 items)
  • Validate links (reject eBay, verify reachability)
  • Confirm color/size options match preferences
  • Suggest similar trending items as alternatives
  • Hand off validated item_list.json to Scraper
Web Search HTTP Client Message Bus
🕷️

Scraper Claw

MVP

The monitoring engine — continuously checks availability, price, and proves it with screenshots.

  • Check item availability via headless browser
  • Extract current price and size matrix
  • Take screenshot proof of each check
  • Detect status changes (unavailable → available)
  • Anti-bot evasion: proxy rotation, user-agent spoofing
  • Compile reports, hand off to Buyer Claw
Playwright ScraperAPI Cloudflare R2 2Captcha
💰

Buyer Claw

Phase 2

The purchasing engine — executes checkouts, manages shipping, and maintains the inventory database.

  • Receive available items from Scraper
  • Get purchase approval (or auto-buy)
  • Execute checkout with stored payment
  • Set shipping to team address
  • Record purchases in inventory DB
  • Hand off to Seller Claw
Playwright Stripe API Supabase Message Bus
📦

Seller Claw

Phase 2

The revenue engine — lists on Poshmark, negotiates offers, and tracks profit.

  • Calculate optimal resale pricing
  • Create Poshmark listings via PrimeLister
  • Monitor and respond to offers automatically
  • Coordinate shipping address changes
  • Track inventory status and profit margins
PrimeLister Poshmark API Supabase Message Bus

Key Talking Points

Why Claws (Multi-Agent)?
Each claw is specialized — the scraper doesn't need to know about pricing strategy, the seller doesn't need to know about anti-bot techniques. Separation of concerns at the AI level.
Why Telegram as Message Bus?
It's both the human interface AND the inter-agent protocol. You can watch the claws "talk" to each other in real-time. Plus it's free and everyone already has it.
Hard Problem: Bot Detection
Many fashion sites block scrapers aggressively. Our approach: residential proxy rotation + Playwright stealth + screenshot proof (which also verifies the scraper actually visited the page).
Revenue Model
Buy items at retail (or below retail on resale platforms), sell at markup on Poshmark. The automation reduces the time overhead from hours to minutes per item, making smaller margins viable at scale.
